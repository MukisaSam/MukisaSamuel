FastAPI · Request Lifecycle

How one request travels through the app

Follow a single request from the browser all the way to the database and back. Each stop is a real folder in the project — the same separation-of-concerns you see in the file tree, now in motion.

GET /api/v1/students/
Request travelling in (down) Response travelling back (up)
1
Client

Browser or API tool

the outside world

Someone opens the page or calls the API. The HTTP request leaves the client and travels across the network to your server.

2
Web server · the waiter

Uvicorn

uvicorn app.main:app

Listens on the port, receives the raw HTTP request, and hands it to your app through the ASGI interface. Nothing reaches FastAPI without going through it.

3
App & routing

FastAPI matches the URL

app/main.py → app/api/v1/router.py

FastAPI looks at the method + path (GET /api/v1/students/) and finds the one endpoint function registered for it.

4
Endpoint

The route function runs

app/api/v1/endpoints/students.py

list_students() is called. It asks for a database session via Depends(get_db), so FastAPI resolves that dependency first.

5
Dependency

Open a database session

app/api/deps.py

get_db() opens a Session, lends it to the endpoint, and guarantees it is closed again when the request finishes.

6
CRUD · the queries

Build the query in Python

app/crud/student.py

crud.student.get_multi(db) describes what to fetch — in Python. No SQL is written by hand here.

7
ORM & model

SQLAlchemy writes the SQL

app/models/student.py

Using the Student table definition, SQLAlchemy translates the Python query into real SQL — the ORM doing its job.

8
Database

SQLite runs the query

students.db

The database executes the SQL and returns the matching rows. This is the turning point — the request has arrived; now the answer heads back.

↑ The response journey back

  • ↑SQLAlchemy turns the raw rows into Student objects your Python code can use.
  • ↑Pydantic shapes the JSON — app/schemas/student.py · StudentRead validates the data and keeps only the public fields.
  • ↑FastAPI serialises those objects to JSON, and Uvicorn sends the HTTP response back out.
  • ↑The client receives the JSON — or, for GET /, the HTML page renders the students in a table.

The one idea to leave them with: every layer has a single job, and the request passes through them in order — routing, validation, queries, the ORM, the database — then the answer retraces its steps home. That is why the code is split into folders instead of one big file.
